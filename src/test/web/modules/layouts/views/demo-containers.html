Imersão na estrutura por trás do Global UX.
Arquitetura principal

As estruturas de interface construídas a partir do GUX seguem uma estrutura principal, cujo alicerce se dá a partir da directiva principal gx-app. Por padrão está diretiva carrega os principais componentes que montam uma tela conforme o diagrama:

bloco obrigatório Descrição padrão
cockpit sim Representa a barra de imersão dos sistemas. Imagine que nesta barra entra tudo o que o usuário precisa ter a mão enquanto navega pelos sistemas. Ex. Troca de senha. true
navbar não Trata-se da barra de navegação inerente ao sistema que está sendo utilizado. true
aside não Bloco lateral que comporta o menu de dois níveis de navegação do sistema. A barra possui nativamente a função de recolhimento. true
main sim Container onde são desenvolvidas as views dos sistemas. Na prática é o bloco mais importante para o desenvolvedor que utiliza o GUX. true
footer não Barra inferior do sistema. Possui apenas função estética. true
Por dentro do gx-app

Temos a directiva gx-app como container principal de toda aplicação que utilize o GUX. Dentro do contexto são aplicadas as directivas dos blocks mostrados acima. Estas directivas possuem uma estrutura padrão no encaixe dos blocos.

gx-app
true (only) gx-cockpit
true / false gx-navbar
true / false gx-aside
true (only) gx-main
true / false gx-footer
