gx-form-button

O framework trás uma lista de directivas para uso de botões. Tais diretivas facilitam a composição dos layouts padrão das telas com formulários.

Modelo Tipos / Método Obrigatório Descrição Marcação HTML
get não Botão pesquisar das telas que utilizam o GUX <gx-form-button stereotype="get"></gx-form-button>
post não Botão utlizado para gravar ações de formulários <gx-form-button stereotype="post"></gx-form-button>
put não Botão utlizado para gravar ações de formulários <gx-form-button stereotype="put"></gx-form-button>
cancel não Botão Cancelar <gx-form-button stereotype="cancel"></gx-form-button>
back não Botão voltar <gx-form-button stereotype="back"></gx-form-button>
clean não Botão limpar, reseta todos os campos do form atribuído. <gx-form-button stereotype="clean"></gx-form-button>

O objetivo destas directivas é facilitar as composições de tela com uso dos padrões do GUX, todavia, é possível utilizar HTML nativo com seletores pré formatados do tema. Veja um exemplo:

Button addon

<button class="btn m-b-xs btn-sm btn-info btn-addon btn-lg"><i class="ion-ios-bolt"></i></button>

No entanto, esta estrutura não permite o uso do atributo "stereotype".
Método POST define o padrão como botão Gravar.
Método PUT define o padrão como botão Atualizar.
Método GET define o padrão como botão Pesquisar.
Método CLEAN(reset) define o padrão como botão Limpar .
Método CANCEL define o padrão como botão Cancelar.
Método BACK define o padrão como botão Voltar.
